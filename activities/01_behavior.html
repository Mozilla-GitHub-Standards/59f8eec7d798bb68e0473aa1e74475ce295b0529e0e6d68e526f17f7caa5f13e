Code Review

What Makes Good Code, Pt. 1: Behavior

The most important trait of any piece of code, is that it gets the job done. In this activity, learners will write & share some code, and try to determine if it's doing what it's supposed to.

Made by the Mozilla Science Lab

Steps for the Activity

  1. Have a look at the project description here - this is an analysis program that consists of four Python functions, each of which is responsible for a single step: data generation, data cleaning, data analysis, and presentation. Randomly choose one of the functions described, and write it yourself. Don't tell anyone which one you chose!

    Now, find a partner and exchange the functions you wrote above - but don't explain them at all! For your partner's function, try and determine the following things:

    • What's it supposed to do? Which of the four functions in the project description is this? How can you tell? What could be done to make it clearer what this function was supposed to do?
    • What's it actually do? Try making up some dummy inputs and running the function. Does it do what you'd expect? Are there inputs you can provide that make it do something different or unexpected?
    • Does it actually do what it's supposed to do? Does the behavior of the function completely satisfy the expectations laid out in the project description? Can you spot times when it will fail?

  2. Share your findings with your partner. Did they understand what you were trying to build? Could it be demonstrated that it did what it was supposed to? Discuss.

  3. Finally, share any surprises or ideas with everyone else. Make a list together of ways to clearly explain what a new piece of code is supposed to do, and strategies you thought of for ensuring that it actually does those things.

Materials

  • Text Editor
  • Web Access
  • Your Friends!

Discussion

After this activity, you can start your own code quality checklist! When you're thinking about a piece of code, start by asking yourself:
  • Is it clear what the code is supposed to do?
  • Does the code actually satisfy its intended purpose?
